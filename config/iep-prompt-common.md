# 通用评估结果 → IEP 生成规则

> 本文档定义所有评估工具通用的 IEP 生成规则和专业标准。
> 各评估工具的专属规则见对应的 iep-prompt-{tool}.md。
> 核心原则：IEP 中的所有干预项目必须来自指定课包的干预项目配置表，不能自行编造。

---

## 1. 干预目标库结构

### 1.1 课包（course_packet）

系统支持多个课包，每个机构使用不同课包：

| 课包 | 项目数 | 说明 |
|------|--------|------|
| RICE1v1 | 316 | RICE AI体系1对1训练课包 |
| 春暖综合ABA | 738 | 南宁春暖综合ABA课程 |
| 春暖言语 | 211 | 广西南宁春暖言语课包 |

IEP 生成时必须指定使用哪个课包，只从该课包中匹配干预项目。

### 1.2 干预项目（intervene_project）

| 字段 | 说明 |
|------|------|
| name | 干预项目名称，如"走台阶""韵母a" |
| territory | 一级领域，如"粗大动作""沟通""认知" |
| subdivide | 二级领域，如"精细动作""语言理解""模仿" |
| age_bracket | 适用年龄段，如"1-2岁""2-3岁" |
| stages[] | 阶段列表，每个阶段有名称和通过标准 |

### 1.3 干预项目阶段（intervene_project_stage）

| 字段 | 说明 |
|------|------|
| name | 阶段名称，如"扶着栏杆上台阶""独自走台阶" |
| stage | 阶段编号，如"L1""L2""L3"或"1""2""3" |
| pass_standard | 通过标准，如 `{"rate": 80, "counts": 3}` = 连续3次正确率≥80% |

### 1.4 RICE1v1 课包领域结构

| territory（一级领域） | subdivide（二级领域） | 项目数 |
|---------------------|---------------------|--------|
| 粗大动作 | 粗大动作、精细动作 | 42 |
| 沟通 | 语言理解、语言表达 | 84 |
| 认知 | 物体恒存、听觉、视觉、配对、概念、模仿、推理、数学、时间 | 54 |
| 社交情绪 | 社交基础、互联注意、同伴互动、情绪、心智理论、社交规则 | 46 |
| 生活自理 | 进食穿衣、家务整理、安全意识、社区技能 | 34 |
| 发音 | 构音、韵母 | 29 |
| 学习行为 | 个训行为、集体行为 | 17 |
| 游戏 | 游戏技能、社会性游戏 | 10 |

## 2. IEP 编写流程

```
Step 1：汇总评估数据
  输入：评估答题数据 + 儿童基本信息（姓名、出生日期、诊断）
  输出：各领域得分率、萌芽/不通过项目清单、行为问题清单

Step 2：确定发展年龄
  输入：各领域得分率 + 儿童生理年龄
  输出：各领域发展年龄（月）
  规则：发展年龄（月）= 得分率 × 生理年龄（月）

Step 3：筛选薄弱领域
  输入：各领域得分率
  输出：需要干预的领域列表（按优先级排序）
  规则：见第 4 节

Step 4：确定干预年龄段
  输入：各领域发展年龄
  输出：每个领域对应的课包年龄段
  规则：见第 6 节

Step 5：匹配干预项目
  输入：薄弱领域 + 干预年龄段 + 课包 + 萌芽项目清单
  输出：匹配到的干预项目列表
  规则：见第 7 节

Step 6：控制数量 + 排序
  输入：匹配到的干预项目
  输出：精简后的 IEP 条目（按优先级排序）
  规则：见第 5 节

Step 7：生成 IEP 表格
  输出：阶梯及领域 / 任务目标 / 短期目标
```

---

## 3. IEP 表格结构

| 列 | 数据来源 | 示例 |
|----|---------|------|
| 阶梯及领域 | `{年龄段}-{territory}-{subdivide}` | `2-3岁-沟通-语言理解` |
| 任务目标 | 干预项目 `name` | `简单句理解` |
| 短期目标 | 干预项目阶段 `stage.name` | `人物+动作/地点` |

---

## 4. 薄弱领域筛选

### 4.1 纳入标准

```
发展技能薄弱 = 得分率 ≤ 65% 的发展领域
行为干预需求 = 行为评级为 S(严重) 或 M(轻微) 的领域
```

### 4.2 优先级排序

```
P0：严重行为问题（自伤、攻击、严重刻板等，影响安全和学习）
P1：得分率 ≤ 35% 且有萌芽技能的领域（最薄弱+最近发展区=干预效果最好）
P2：得分率 ≤ 35% 的领域（最薄弱）
P3：得分率 36%~50% 且有萌芽技能的领域
P4：得分率 36%~50% 的领域
P5：得分率 51%~65% 的领域
P6：轻度行为问题
```

## 5. IEP 项目数量控制

### 5.1 领域和目标数量

| 项目 | 推荐 | 上限 |
|------|------|------|
| 干预领域数 | 3~5 个 | 6 个 |
| 每领域任务目标 | 2~4 个 | 5 个 |
| 每目标短期阶段 | 1~3 个 | 全部阶段 |
| 整份 IEP 总目标 | 8~15 个 | 20 个 |

### 5.2 按发展年龄调整

| 发展年龄 | 领域数 | 总目标数 | 说明 |
|---------|--------|---------|------|
| < 2岁 | 2~3 | 6~8 | 低龄注意力短，目标少而集中 |
| 2~3岁 | 3~4 | 8~12 | 逐步增加覆盖 |
| 3~5岁 | 4~5 | 10~15 | 学龄前关键期 |
| > 5岁 | 3~5 | 8~12 | 聚焦社交、学业准备、生活自理 |

### 5.3 特殊情况

**全面落后（所有领域 ≤ 35%）**：
```
优先选 3 个基础领域：沟通 + 认知(模仿) + 社交情绪
原因：沟通和模仿是学习前提，社交是融入环境基础
其余领域下个 IEP 周期再纳入
```

**能力不均衡（强弱差距 > 30%）**：
```
弱势领域：正常纳入 IEP
强势领域：不纳入 IEP，但标注"可作为教学媒介"
利用强项带动弱项（如视觉强→用视觉支持教语言）
```

## 6. 年龄段匹配

### 6.1 核心原则

**按发展年龄选课包年龄段，不按生理年龄。**

### 6.2 映射表

| 领域发展年龄（月） | 课包年龄段 |
|-----------------|---------|
| 0~23 | 1-2岁 |
| 24~35 | 2-3岁 |
| 36~47 | 3-4岁 |
| 48~59 | 4-5岁 |
| ≥ 60 | 5岁以上 |

### 6.3 边界处理

发展年龄在边界附近（如23~25月）时：
```
同时查看相邻两个年龄段
优先选较低年龄段中未掌握的项目（确保基础扎实）
再补充较高年龄段中与萌芽技能匹配的项目
```

---

## 7. 干预项目匹配规则

### 7.1 匹配流程

```
对每个薄弱领域：
  1. 通过领域映射规则（见各工具专属prompt）→ 按语义匹配和能力维度匹配，从课包中找到对应的 territory + subdivide
  2. 筛选：该 territory + subdivide + 干预年龄段 的所有干预项目
  3. 排序匹配：
     a. 与萌芽技能直接对应的干预项目（最优先）
     b. 与不通过技能对应的干预项目中，阶段最低的
     c. 该领域该年龄段的其他干预项目（补充）
  4. 取前 2~4 个作为该领域的任务目标
```

### 7.2 阶段选择

```
每个干预项目的阶段（stage）按编号顺序排列：
  → 短期目标 = 从阶段 1 开始列出
  → 如果评估显示儿童已具备某阶段能力，可跳过
  → 每个任务目标列 1~3 个阶段
```

### 7.3 匹配失败

```
课包中找不到对应干预项目时：
  → 跳过，不生成 IEP 条目
  → 在备注中标注"XX领域建议补充专项干预方案"
  → 绝对不能编造不存在的项目名称
```

---

## 8. IEP 周期和评估

```
IEP 周期：3 个月（一个评估周期）
短期目标预期：每个目标 2~4 周达成
评估频率：每月评估一次短期目标达成情况
通过标准：按干预项目阶段的 pass_standard 执行
```

---

## 9. 输出格式

### 9.1 IEP 表格

按优先级从高到低排列：

```
| 阶梯及领域 | 任务目标 | 短期目标 |
|-----------|---------|---------|
| {年龄段}-{territory}-{subdivide} | {project.name} | {stage.name} |
```

### 9.2 导出文件

```
文件名：{姓名}个别化教育计划.docx
内容：封面 + 评估摘要 + IEP 表格 + 备注
```

---

## 10. 确定性要求

- 同一份评估数据 + 同一个课包 → 同一份 IEP
- 所有项目名称和阶段名称必须与课包完全一致
- 不能编造、改写或合并课包中的项目
