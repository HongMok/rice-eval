# PEP-3 评估报告 — 计算规则

> 本文档定义 PEP-3 评估报告的所有计算规则。
> 同一份评估数据，必须产出完全相同的报告结果。

## 3. 副测验分数计算

### 3.1 原积分（Raw Score）

```
原积分 = 该副测验中得分为 2（通过/恰当）的题目数量
注意：只计 2 分的题目数，不是总得分。
```

### 3.2 发展年龄（Developmental Age）

通过原积分查**发展表现图常模表**得到发展年龄（月数）。

```
查表方法：
1. 取该副测验的原积分
2. 在常模表的该副测验列中，找到原积分对应的年龄行
3. 原积分恰好在表中 → 直接取该行年龄
4. 原积分在两行之间 → 取较低年龄行（向下取整）
5. 发展年龄 = 查到的年龄（月）
```

**简化公式**（无常模表时）：
```
发展年龄（月）= (原积分 / 该副测验总题数) × 儿童实足年龄（月）
```

适用范围：
- 发展副测验（CVP, EL, RL, FM, GM, VMI）：有发展年龄
- 照顾者报告-PSC：有发展年龄
- 行为副测验（AE, SR, CMB, CVB）和 PB、AB：无发展年龄，只有适应程度

### 3.3 百分比级数（Percentile Rank）

通过原积分查常模表。简化映射：

```
得分率 = 原积分 / 该副测验总题数 × 100%
```

| 得分率 | 百分比级数 |
|--------|-----------|
| ≥ 90% | 95 |
| 80%~89% | 75 |
| 70%~79% | 50 |
| 60%~69% | 25 |
| 50%~59% | 10 |
| < 50% | 5 |

### 3.4 发展/适应程度

| 百分比级数 | 程度 | 显示 |
|-----------|------|------|
| ≥ 75 | 适当 | 绿色标签 |
| 25 ~ 74 | 轻度 | 黄色标签 |
| < 25 | 严重 | 红色标签 |

### 3.5 副测验分数表格

报告中展示为一张表格：

## 4. 合成分数计算

### 4.1 三个合成分数

| 合成分数 | 代码 | 组成副测验 |
|---------|------|-----------|
| 沟通 | C | CVP + EL + RL |
| 体能 | M | FM + GM + VMI |
| 行为 | MB | AE + SR + CMB + CVB |

### 4.2 计算步骤

```
Step 1: 取各组成副测验的百分比级数
Step 2: 标准分总和 = 各组成副测验的百分比级数之和
Step 3: 合成百分比级数 = 标准分总和 / 组成副测验数（简化）
Step 4: 合成发展/适应程度 = 同 3.4 的映射规则
Step 5: 合成发展年龄 = 各组成副测验发展年龄的平均值（仅沟通C和体能M有）
```

### 4.3 合成分数表格

```
                标准分 Standard Scores (SSs)
                CVP  EL  RL  FM  GM  VMI  AE  SR  CMB  CVB  标准分总和  百分比级数  发展/适应程度  发展年龄
沟通(C)          {分} {分} {分}                                    {和}      {X%}      {程度}       {X月}
体能(M)                        {分} {分} {分}                      {和}      {X%}      {程度}       {X月}
行为(MB)                                      {分} {分} {分} {分}  {和}      {X%}      {程度}       —
```

## 5. 发展表现图（常模侧面图）

### 5.1 图表结构

- X轴 = 7列：CVP、EL、RL、FM、GM、VMI、PSC
- Y轴 = 年龄（月），范围 <12 ~ 83月
- 背景 = 常模表数值（粉色条纹行）

### 5.2 两条数据线

**● 通过项目分数（P分，深绿色实线）**

```
每个副测验的 P 值 = 该副测验中得分为 2 的题目数
在常模表中找到 P 值对应的年龄行 → 画点连线
```

**■ 部分通过项目分数（P+E分，浅绿色实线）**

```
每个副测验的 P+E 值 = 得分为 2 的题目数 + 得分为 1 的题目数
在常模表中找到 P+E 值对应的年龄行 → 画点连线
```

### 5.3 P+E 与 P 的差距 = 萌芽技能

```
P+E 线与 P 线之间的差距 = 萌芽技能数量
差距越大 → 该领域的最近发展区越宽 → 干预潜力越大
```

### 5.4 底部汇总

```
通过项目分数：{各副测验P值}
部分通过项目分数：{各副测验P+E值}
```

## 6. 发展数据分析（水平柱状图）

### 6.1 图表结构

- 6个发展副测验，按得分率从低到高排列（最弱在最上面）
- 每个副测验一行水平柱状图
- 深蓝色 = P（通过）占比
- 浅蓝色 = E（萌芽）占比
- 红色虚线 = 65% 薄弱线（在最上层，不被柱子遮挡）

### 6.2 得分率计算

```
得分率 = P题数 / 总题数 × 100%

只计算完全通过(P=2分)的题目占比。
萌芽(E)不计入得分率，但在柱状图中以浅蓝色展示，作为"最近发展区"的可视化。
```

柱状图中：
- 深蓝柱宽度 = P / 总题数（与得分率百分比完全对应）
- 浅蓝柱宽度 = E / 总题数（紧接深蓝柱后面，展示萌芽空间）
- 两段柱子加起来 = (P+E) / 总题数

### 6.3 颜色标注

```
得分率 ≤ 65%：领域名和百分比显示为红色（薄弱，需干预）
得分率 > 65%：领域名和百分比显示为绿色（达标）
```

### 6.4 右侧标注

```
每行右侧显示：
  {百分比}%
  通过{P数} 萌芽{E数} 不通过{F数}
```

### 6.5 图例

```
■ 通过(P)    ■ 萌芽(E)    --- 65% 薄弱线
```

## 7. 行为数据分析（堆叠柱状图）

### 7.1 图表结构

- 4个行为副测验，每个一行水平堆叠柱状图
- 绿色 = A（恰当）占比
- 黄色 = M（轻微）占比
- 红色 = S（严重）占比

### 7.2 A/M/S 统计

```
对每个行为副测验：
  A数 = 得分为 2（恰当）的题目数
  M数 = 得分为 1（轻微）的题目数
  S数 = 得分为 0（严重）的题目数
```

### 7.3 右侧标注

```
每行右侧显示：A{数} M{数} S{数}
```

### 7.4 图例

```
■ A 恰当    ■ M 轻微    ■ S 严重
```

## 8. 综合分析生成规则

### 8.1 各领域得分概述

```
格式：
本次评估涵盖功能发展{6}个领域和行为表现{4}个领域。
功能发展总分{总题数}分中，已有能力(P){P总数}分，部分能力(E){E总数}分，
未达到(F){F总数}分。整体发展年龄约{X}岁{Y}月。
```

### 8.2 优势领域

```
列出所有得分率 ≥ 65% 的发展副测验，按得分率降序
格式：{领域名}({得分率}%)
示例：知觉(82%)、认知理解(68%)，为相对较好的领域。
```

### 8.3 需关注领域

```
列出所有得分率 ≤ 65% 的发展副测验，按得分率升序
格式：{领域名}({得分率}%)，得分率偏低，建议作为重点干预领域。
```

### 8.4 行为表现

```
列出所有行为副测验的评级
格式：{领域名}({评级}-{含义})
示例：情感关系(M-轻度)、游戏及兴趣(A-适当)、感觉反应(M-轻度)、语言异常(M-轻度)。
```

### 8.5 建议

```
优先级：
P1：S级行为副测验 → "优先干预{领域名}等严重行为问题"
P2：得分率最低的发展副测验 → "重点加强{领域名}等薄弱领域的训练"
P3：M级行为副测验 → "关注{领域名}等轻度行为问题"
P4：通用建议 → "家庭中可增加亲子互动游戏，鼓励孩子主动表达需求"
```

## 9. 确定性要求

- 同一份输入数据 → 同一份报告输出，无例外
- 不使用随机措辞、不使用"可能""大概"等模糊表述
- 所有数值计算使用四舍五入
- 所有阈值判断使用闭区间（含边界值）
- 原积分只计 2 分（通过/恰当）的题目数，不是总得分
- 发展数据分析图表中，≤65% 标红，>65% 标绿
- 薄弱线（65%红色虚线）在最上层，不被柱子遮挡
