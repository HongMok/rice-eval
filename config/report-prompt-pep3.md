## 生成规则（量化标准，确保同一份数据只生成一份确定性报告）

### 功能发展评分量化规则

1. **得分率** = P分 / 总题数 × 100%，四舍五入取整
2. **领域评价**（基于得分率，确定性映射）：
   - 得分率 ≥ 75% → "强处"
   - 得分率 ≤ 50% → "弱处"
   - 其余 → 空（不标注）
3. **发展年龄推算**（基于得分率，确定性映射）：
   - 得分率 ≥ 90% → 实足年龄
   - 得分率 80%~89% → 实足年龄 - 6个月
   - 得分率 70%~79% → 实足年龄 - 12个月
   - 得分率 60%~69% → 实足年龄 - 18个月
   - 得分率 50%~59% → 实足年龄 - 24个月
   - 得分率 40%~49% → 实足年龄 - 30个月
   - 得分率 < 40% → 实足年龄 - 36个月
4. **整体发展年龄** = 7个领域发展年龄的中位数

### 行为表现评分量化规则

1. **A/M/S 评级**（基于平均分，确定性映射）：
   - 平均分 ≥ 1.5 → A（适当）
   - 平均分 0.8~1.49 → M（轻度）
   - 平均分 < 0.8 → S（严重）
2. 直接使用输入数据中的 rating，不重新计算

### 副测验分数表量化规则（第2部分）

1. **原始分** = P分（通过的题目数）
2. **发展年龄** = 按上述发展年龄推算规则计算
3. **百分比级数**（基于得分率，确定性映射）：
   - 得分率 ≥ 90% → 95
   - 得分率 80%~89% → 75
   - 得分率 70%~79% → 50
   - 得分率 60%~69% → 25
   - 得分率 50%~59% → 10
   - 得分率 < 50% → 5
4. **发展/适应程度**（基于百分比级数，确定性映射）：
   - 百分比级数 ≥ 75 → "适当"
   - 百分比级数 50~74 → "轻度"
   - 百分比级数 < 50 → "严重"

### 合成分数表量化规则（第3部分）

1. **沟通(C)** = CVP + EL + RL 三个子测验的标准分总和
2. **体能(M)** = FM + GM + VMI 三个子测验的标准分总和
3. **行为(MB)** = AE + SR + CMB + CVB 四个子测验的标准分总和
4. **标准分** = 百分比级数（简化处理，实际应查常模表）
5. **合成百分比级数** = 合成标准分总和 / 合成满分 × 100，映射到百分比级数
6. **合成发展/适应程度** = 同副测验的映射规则

### 教育训练纲要量化规则

1. `eduOutline` 必须包含 3~5 条建议
2. 每条建议必须对应一个具体的薄弱领域（得分率 ≤ 65% 或行为评级 S/M）
3. 建议按优先级排序：S级行为 > 弱处领域 > M级行为 > 中等领域

### 综合分析量化规则

1. `analysis.overview`：必须包含功能发展总分（P/E/F）和行为表现评级
2. `analysis.strengths`：列出所有"强处"领域（得分率 ≥ 75%）
3. `analysis.concerns`：列出所有"弱处"领域（得分率 ≤ 50%）+ S级行为领域
4. `analysis.recommendations`：基于以下优先级排序：
   - P1：S级行为领域（优先干预严重行为问题）
   - P2：弱处功能发展领域（优先提升最薄弱技能）
   - P3：M级行为领域（关注轻度行为问题）
5. **确定性要求**：同一份输入数据，必须生成完全相同的报告内容。不使用随机措辞，所有判断基于上述量化阈值
