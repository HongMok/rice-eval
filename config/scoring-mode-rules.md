# 评估答题 — 统计模式与评分规则

## 两种统计模式

### 1. 直接评分模式（默认）

评估师直接点击评分等级按钮（如 0-不通过 / 1-萌芽 / 2-通过），一次点击完成评分。

适用场景：评估师对该题目已有明确判断，不需要多回合探测。

### 2. 回合统计模式（Cold Probe）

评估师对每道题进行多回合探测（默认 3 回合，可自由增加），每回合记录三种结果之一：
- **独立完成 (+)**：儿童在无辅助下独立完成
- **错误 (-)**：儿童未能完成或回答错误
- **辅助完成 (P)**：儿童在辅助（手势、口头、示范、肢体等）下完成

系统根据回合统计结果自动换算出最终评分。

## 回合统计 → 评分换算规则

### 通用换算逻辑

根据独立完成率（独立完成回合数 / 总回合数）换算：

| 独立完成率 | 换算评分 | 说明 |
|---------|--------|------|
| ≥ 80%   | 最高分  | 稳定掌握（如 PEP-3 的 2-通过，VB-MAPP 的 1-达标） |
| 30%~79% | 中间分  | 萌芽/部分达标（如 PEP-3 的 1-萌芽，VB-MAPP 的 0.5-部分达标） |
| < 30%   | 最低分  | 未掌握（如 PEP-3 的 0-不通过，VB-MAPP 的 0-未达标） |

### 辅助完成的处理

辅助完成 (P) 不计入独立完成，但会影响换算：
- 如果全部回合都是辅助完成（独立0%，辅助100%），评为中间分（而非最低分），因为有辅助下能完成说明技能处于萌芽状态
- 如果全部回合都是错误（独立0%，辅助0%），评为最低分

### 换算公式

```
独立率 = 独立完成数 / 总回合数
辅助率 = 辅助完成数 / 总回合数
错误率 = 错误数 / 总回合数

if (独立率 >= 0.8) → 最高分
else if (独立率 >= 0.3) → 中间分
else if (辅助率 > 0) → 中间分（有辅助能力）
else → 最低分
```

### 各工具的评分映射

| 工具 | 最高分 | 中间分 | 最低分 |
|------|--------|--------|--------|
| PEP-3 | 2 (通过) | 1 (萌芽) | 0 (不通过) |
| C-PEP | 1 (通过) | 0.5 (中间反应) | 0 (不通过) |
| VB-MAPP | 1 (达标) | 0.5 (部分达标) | 0 (未达标) |

### 最少回合数

- 最少 1 个回合即可出分（单回合时：独立→最高分，辅助→中间分，错误→最低分）
- 建议 3 个回合以获得更稳定的评估结果
- 用户可自由增加回合数，无上限

## 模式切换的边界处理

### 从直接评分 → 回合统计

- 如果该题已有直接评分结果，弹出确认弹窗：
  - "切换到回合统计模式后，当前评分将被清除，需要重新通过回合统计获得评分。是否继续？"
  - 确认后：清除该题的直接评分，进入回合统计模式（空白回合）
  - 取消：保持直接评分模式

### 从回合统计 → 直接评分

- 如果该题已有回合统计数据，弹出确认弹窗：
  - "切换到直接评分模式后，回合统计数据将被清除。是否继续？"
  - 确认后：清除回合数据，进入直接评分模式
  - 取消：保持回合统计模式

### 切换范围

- 统计模式是全局设置（整份评估统一），不是逐题设置
- 切换时影响所有已答题目，需要明确提示

## 数据存储

回合统计数据存储在 localStorage 的答题数据中：

```json
{
  "pep3_q1": {
    "value": 2,
    "label": "通过",
    "scoringMode": "rounds",
    "rounds": [
      { "round": 1, "result": "independent" },
      { "round": 2, "result": "error" },
      { "round": 3, "result": "prompted" }
    ],
    "independentRate": 0.33,
    "notes": "第一回合自发完成，后两回合注意力下降"
  }
}
```

直接评分模式的数据格式不变：
```json
{
  "pep3_q1": {
    "value": 2,
    "label": "通过",
    "scoringMode": "direct"
  }
}
```

## UI 设计要点

### 设置按钮
- 位置：答题页右上角（进度条右侧）
- 图标：SVG 齿轮图标（不用 emoji）
- 点击弹出浮窗

### 设置浮窗
- 统计方式切换：两个选项卡（直接评分 / 回合统计）
- 当前选中项高亮
- 切换时如有数据则弹出确认

### 回合统计 UI（替换原有评分按钮区域）
- 标题行："冷测 (Cold Probe)" + 右侧说明"默认建议 3 回合，可自由增加"
- 每回合一行：回合标签（第1回合/第2回合...）+ 三个按钮（独立(+) 绿色 / 错误(-) 红色 / 辅助(P) 橙色）
- 底部："+ 新增测试回合"虚线按钮
- 回合统计完成后，自动显示换算结果（如"换算评分：1-萌芽（独立率33%）"）
