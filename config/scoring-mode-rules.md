# 评估答题 — 统计模式与评分规则

## 两种统计模式

### 1. 直接评分模式（默认）

评估师直接点击评分等级按钮，一次点击完成评分。

按钮显示格式：`X分（标签）`，如 `1分（达标）`、`0分（错误）`。无标签时只显示 `X分`。

评分按钮规则：
- **过滤**：评分分值为空的等级不展示
- **排序**：按分值从低到高排列（左→右）
- **颜色**：最低分=红色，中间分=橙色，最高分=绿色

适用场景：评估师对该题目已有明确判断，不需要多回合探测。

### 2. 回合统计模式（Cold Probe）

评估师对每道题进行多回合探测（默认 3 回合，可自由增加），每回合记录三种结果之一：
- **独立完成 (+)**：儿童在无辅助下独立完成
- **错误 (-)**：儿童未能完成或回答错误
- **辅助完成 (P)**：儿童在辅助（手势、口头、示范、肢体等）下完成

系统根据回合统计结果自动换算出最终评分。

## 回合统计 → 评分换算规则

### 核心思路：百分比线性映射

用独立率（独立完成数 / 总回合数）作为百分比，线性映射到该题的分值区间 [最低分, 最高分]，然后四舍五入到最近的有效评分等级。

这种方式天然兼容任意数量的评分等级（2级～10级均适用）。

### 换算公式

```
独立率 = 独立完成数 / 已答回合数
辅助率 = 辅助完成数 / 已答回合数
等级数 = 该题的评分等级数量（2~10）

// Step 1: 计算有效比率 effectiveRate（0~1）
if (独立率 >= 80%)        → effectiveRate = 1.0（稳定掌握）
else if (全部错误)         → effectiveRate = 0（完全未掌握）
else if (独立=0 且 辅助>0) → effectiveRate = 等级数 <= 2 ? 0.5 : 1/(等级数-1)
                            // 2级时映射到中间(0.5)，3级以上映射到第2等级附近
else                       → effectiveRate = 独立率（线性映射）

// Step 2: 线性插值到分值
rawValue = 最低分 + effectiveRate × (最高分 - 最低分)

// Step 3: 四舍五入到最近的有效评分等级
resultScore = 距离 rawValue 最近的评分等级
```

### 映射示例

**2级评分（如 H/L：0, 1）**

| 独立率 | effectiveRate | rawValue | 映射结果 |
|--------|--------------|----------|----------|
| 100% (3/3独立) | 1.0 | 1.0 | 1分（通过） |
| 80% (4/5独立) | 1.0 | 1.0 | 1分（通过） |
| 67% (2/3独立) | 0.67 | 0.67 | 1分（通过） |
| 33% (1/3独立) | 0.33 | 0.33 | 0分（不通过） |
| 0% (全辅助) | 0.5 | 0.5 | 1分（通过）* |
| 0% (全错误) | 0 | 0 | 0分（不通过） |

*注：2级评分中全辅助映射到0.5，四舍五入到1分（通过）。设计意图：有辅助下能完成说明具备基础能力。

**3级评分（VB-MAPP 里程碑：0, 0.5, 1）**

| 独立率 | effectiveRate | rawValue | 映射结果 |
|--------|--------------|----------|----------|
| 100% (3/3独立) | 1.0 | 1.0 | 1分（达标） |
| 67% (2/3独立) | 0.67 | 0.67 | 0.5分（部分达标） |
| 33% (1/3独立) | 0.33 | 0.33 | 0.5分（部分达标） |
| 0% (全辅助) | 0.5 | 0.5 | 0.5分（部分达标） |
| 0% (全错误) | 0 | 0 | 0分（错误） |

**5级评分（VB-MAPP 转衔：1, 2, 3, 4, 5）**

| 独立率 | effectiveRate | rawValue | 映射结果 |
|--------|--------------|----------|----------|
| 100% (3/3独立) | 1.0 | 5.0 | 5分 |
| 80% (4/5独立) | 1.0 | 5.0 | 5分 |
| 67% (2/3独立) | 0.67 | 3.68 | 4分 |
| 33% (1/3独立) | 0.33 | 2.32 | 2分 |
| 0% (全辅助) | 0.25 | 2.0 | 2分 |
| 0% (全错误) | 0 | 1.0 | 1分 |

**4级评分（VB-MAPP 障碍：1, 2, 3, 4）**

| 独立率 | effectiveRate | rawValue | 映射结果 |
|--------|--------------|----------|----------|
| 100% | 1.0 | 4.0 | 4分 |
| 67% | 0.67 | 3.01 | 3分 |
| 33% | 0.33 | 1.99 | 2分 |
| 0% (全辅助) | 0.33 | 1.99 | 2分 |
| 0% (全错误) | 0 | 1.0 | 1分 |

### 辅助完成的特殊处理

辅助完成 (P) 不计入独立完成，但不等于错误：
- **全辅助无独立**：effectiveRate = `等级数<=2 ? 0.5 : 1/(等级数-1)`
  - 2级 → effectiveRate=0.5 → 四舍五入到较高分（有辅助=有基础能力）
  - 3级 → effectiveRate=0.5 → 中间分
  - 4级 → effectiveRate=0.33 → 第2等级
  - 5级 → effectiveRate=0.25 → 第2等级
  - 10级 → effectiveRate=0.11 → 第2等级
- **全错误无辅助**：effectiveRate = 0 → 最低分
- 设计意图：有辅助下能完成说明技能处于萌芽状态，应高于完全不会

### 最少回合数

- 最少 1 个回合即可出分
- 建议 3 个回合以获得更稳定的评估结果
- 用户可自由增加回合数，无上限

## 模式切换的边界处理

### 从直接评分 → 回合统计

- 如果该题已有直接评分结果，弹出确认弹窗：
  - "切换到回合统计模式后，当前评分将被清除，需要重新通过回合统计获得评分。是否继续？"
  - 确认后：清除该题的直接评分，进入回合统计模式（空白回合）
  - 取消：保持直接评分模式

### 从回合统计 → 直接评分

- 如果该题已有回合统计数据，弹出确认弹窗：
  - "切换到直接评分模式后，回合统计数据将被清除。是否继续？"
  - 确认后：清除回合数据，进入直接评分模式
  - 取消：保持回合统计模式

### 切换范围

- 统计模式是全局设置（整份评估统一），不是逐题设置
- 切换时影响所有已答题目，需要明确提示

## 数据存储

回合统计数据存储在 localStorage 的答题数据中：

```json
{
  "pep3_q1": {
    "value": 2,
    "label": "通过",
    "scoringMode": "rounds",
    "rounds": [
      { "round": 1, "result": "independent" },
      { "round": 2, "result": "error" },
      { "round": 3, "result": "prompted" }
    ],
    "independentRate": 0.33,
    "notes": "第一回合自发完成，后两回合注意力下降"
  }
}
```

直接评分模式的数据格式不变：
```json
{
  "pep3_q1": {
    "value": 2,
    "label": "通过",
    "scoringMode": "direct"
  }
}
```

## UI 设计要点

### 评分按钮显示格式

- 有标签：`X分（标签）`，如 `1分（达标）`、`0.5分（4个）`
- 无标签：`X分`，如 `1分`、`2分`

### 设置按钮
- 位置：答题页右上角（进度条右侧）
- 图标：SVG 齿轮图标（不用 emoji）
- 点击弹出浮窗

### 设置浮窗
- 统计方式切换：两个选项卡（直接评分 / 回合统计）
- 当前选中项高亮
- 切换时如有数据则弹出确认

### 回合统计 UI（替换原有评分按钮区域）
- 标题行："回合统计" + 右侧说明"默认建议 3 回合，可自由增加"
- 每回合一行：回合标签（第1回合/第2回合...）+ 三个按钮（独立(+) 绿色 / 错误(-) 红色 / 辅助(P) 橙色）
- 底部："+ 新增测试回合"虚线按钮 + "重置"按钮
- 回合统计完成后，自动显示换算结果（如"换算评分：1分（萌芽）（独立率33%）"）
