# VB-MAPP 评估结果 → IEP 生成规则

> 本文档定义 VB-MAPP 评估结果如何转化为个别化教育计划（IEP）。
> 核心原则：IEP 中的所有内容必须来自课包干预项目配置表，不能自行编造。

---

## 1. IEP 表格结构

同 PEP-3，三列：阶梯及领域、任务目标、短期目标。

---

## 2. 薄弱领域筛选

### 2.1 阈值

```
薄弱领域 = 里程碑得分率 ≤ 50% 的领域
```

### 2.2 排序

按得分率升序排列。

### 2.3 障碍纳入

障碍评估中 `topConcerns`（得分 ≥ 3）的障碍项，如果课包中有对应干预项目也应纳入 IEP。

---

## 3. 领域映射

VB-MAPP 评估领域 → 课包干预领域：

| VB-MAPP 领域 | 课包领域 |
|-------------|---------|
| 提要求(Mand) | Mand / 沟通-语言表达 |
| 命名(Tact) | Tact / 沟通-语言表达 |
| 听者反应(Listener) | 沟通-语言理解 |
| 视觉配对(VP/MTS) | 认知-配对 |
| 游戏(Play) | 社交情绪-游戏 |
| 社交(Social) | 社交情绪-社交 |
| 模仿(Imitation) | 模仿 |
| 仿说(Echoic) | 发音-仿说 |
| LRFFC | 沟通-语言理解 |
| 对话(Intraverbal) | 沟通-对话 |

---

## 4. 生成规则

1. **任务目标 = 干预项目名称**，颗粒度细到具体项目
2. **短期目标 = 干预项目阶段名称**，按阶段顺序排列
3. **VB-MAPP 特有**：遵循 Sundberg 建议，优先提升 Mand（提要求）
4. **不匹配时跳过**，**不能编造**
5. **导出文件命名**：`{姓名}个别化教育计划.docx`

---

## 5. 优先级排序

```
P1：Mand（提要求）领域 — Sundberg 建议优先提升
P2：障碍得分 ≥ 3 的项目对应的干预项目
P3：里程碑得分率 < 40% 的领域
P4：里程碑得分率 40%~50% 的领域
P5：转衔得分 ≤ 1 的领域
```
