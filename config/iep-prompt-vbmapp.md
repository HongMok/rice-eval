# VB-MAPP 评估结果 → IEP 生成规则

> 本文档定义 VB-MAPP 评估结果如何转化为个别化教育计划（IEP）。
> 核心原则：IEP 中的所有干预项目必须来自指定课包的干预项目配置表，不能自行编造。

---

## 1. 评估数据结构

VB-MAPP 评估数据包含三部分：

| 板块 | 说明 | 评分体系 | IEP 用途 |
|------|------|---------|---------|
| 里程碑评估 | 170个里程碑，分3个等级 | 0/0.5/1分 | 未达标项 = 干预目标 |
| 障碍评估 | 24个障碍领域 | 0~4分(越高越严重) | 高分项 = 需优先处理的障碍 |
| 转衔评估 | 18个转衔技能 | 0~5分(越高越好) | 低分项 = 转衔准备不足 |

## 3. IEP 编写流程

```
Step 1：汇总评估数据
  输入：VB-MAPP 答题数据 + 儿童基本信息
  输出：各领域得分率、0.5分项清单、障碍评估高分项、转衔低分项

Step 2：确定当前等级和干预年龄段（见第9节）
Step 3：筛选薄弱领域（见第5节）
Step 4：匹配干预项目（见第10节）
Step 5：控制数量（见第8节）
Step 6：生成 IEP 表格
```

## 4. IEP 表格结构

| 列 | 数据来源 | 示例 |
|----|---------|------|
| 阶梯及领域 | `{年龄段}-{territory}-{subdivide}` | `1-2岁-沟通-语言表达` |
| 任务目标 | 干预项目 `name` | `提要求-基础` |
| 短期目标 | 干预项目阶段 `stage.name` | `用单词提要求` |

---

## 5. 薄弱领域筛选

### 5.1 里程碑评估

```
领域得分率 = 领域总得分 / 领域满分 × 100%
薄弱领域 = 得分率 ≤ 50%（VB-MAPP 阈值比 PEP-3/C-PEP-3 低，因为评分更细）
```

### 5.2 障碍评估

```
严重障碍 = 得分 ≥ 3（必须在 IEP 中专门处理）
中度障碍 = 得分 = 2（需关注但不一定单独列入）
```

### 5.3 转衔评估

```
转衔不足 = 得分 ≤ 1（如果儿童即将进入新环境，需纳入 IEP）
```

---

## 6. 优先级排序（VB-MAPP 特有）

VB-MAPP 基于 Skinner 的语言行为理论，优先级有其独特逻辑：

```
P0：障碍评估得分 ≥ 3 的项目（严重障碍必须优先处理）
    → 常见：逃避/回避行为、自伤行为、刻板行为、注意力缺陷

P1：提要求(Mand)领域（Sundberg 核心建议）
    → 提要求是语言行为的基础，是儿童主动沟通的起点
    → 即使其他领域更弱，Mand 也应优先或同步干预
    → 原因：Mand 直接受动机控制，训练效果最快，且能减少行为问题

P2：里程碑得分率 ≤ 30% 且处于当前等级的领域

P3：里程碑得分率 31%~50% 的领域

P4：模仿(Imitation)和仿说(Echoic)领域
    → 模仿是学习其他技能的前提

P5：转衔评估得分 ≤ 1 的领域（仅当儿童即将转衔时）

P6：障碍评估得分 = 2 的项目
```

## 8. IEP 项目数量控制

### 8.1 领域和目标数量

| 项目 | 推荐 | 上限 |
|------|------|------|
| 干预领域数 | 3~5 个 | 6 个 |
| 每领域任务目标 | 2~4 个 | 5 个 |
| 每目标短期阶段 | 1~3 个 | 全部阶段 |
| 整份 IEP 总目标 | 8~15 个 | 20 个 |

原则：宁可少而精，不可多而散。

### 8.2 按发展年龄调整

| 发展年龄 | 领域数 | 总目标数 | 说明 |
|---------|--------|---------|------|
| < 2岁 | 2~3 | 6~8 | 低龄注意力短，目标少而集中 |
| 2~3岁 | 3~4 | 8~12 | 逐步增加覆盖 |
| 3~5岁 | 4~5 | 10~15 | 学龄前关键期 |
| > 5岁 | 3~5 | 8~12 | 聚焦社交、学业准备、生活自理 |

### 8.3 特殊情况

**全面落后**：优先选 Mand(提要求) + 模仿 + 听者反应 3个基础领域（Sundberg建议）。

**能力不均衡**：弱势领域正常纳入，强势领域标注"可作为教学媒介"。

## 9. 年龄段匹配（VB-MAPP 特有）

VB-MAPP 用里程碑等级定位，不直接算发展年龄：

| VB-MAPP 等级 | 发展年龄范围 | 课包年龄段 |
|-------------|-----------|---------|
| 1级(里程碑1-5) | 0~18个月 | 1-2岁 |
| 2级(里程碑6-10) | 18~30个月 | 2-3岁 |
| 3级(里程碑11-15) | 30~48个月 | 3-4岁 |

```
干预年龄段 = 当前等级对应的课包年龄段
当前等级 = 该领域最高通过的里程碑所在等级
如果当前等级里程碑 ≥ 70% 通过，可进入下一等级年龄段
```

## 10. 干预项目匹配规则

### 10.1 匹配流程

```
对每个薄弱领域：
  1. 通过领域映射表（第7节）→ 得到课包 territory + subdivide
  2. 筛选：该 territory + subdivide + 干预年龄段 的所有干预项目
  3. 排序匹配：
     a. 与 0.5 分技能直接对应的干预项目（最优先，类似萌芽）
     b. 与 0 分技能对应的干预项目中，阶段最低的
     c. 该领域该年龄段的其他干预项目（补充）
  4. 取前 2~4 个作为该领域的任务目标
```

### 10.2 阶段选择

```
每个干预项目的阶段按编号顺序排列：
  → 短期目标 = 从阶段 1 开始列出
  → 如果评估显示儿童已具备某阶段能力，可跳过
  → 每个任务目标列 1~3 个阶段
```

### 10.3 匹配失败

```
课包中找不到对应干预项目时：
  → 跳过，不生成 IEP 条目
  → 在备注中标注"XX领域建议补充专项干预方案"
  → 绝对不能编造不存在的项目名称
```

## 11. 障碍评估 → IEP 转化

```
障碍得分 ≥ 3（严重）：
  → 必须在 IEP 中设立专门的行为干预目标
  → 匹配课包"学习行为"领域的干预项目
  → 课包无对应项目时，备注"建议制定专项行为干预方案(BIP)"

障碍得分 = 2（中度）：
  → 在相关技能领域的 IEP 目标中附加行为支持策略
  → 不单独列为 IEP 条目

障碍得分 ≤ 1：不纳入 IEP
```

---

## 12. 0.5分项处理

VB-MAPP 的 0.5 分类似于 PEP-3 的"萌芽"：

```
0.5 分 = 儿童部分达标（如：能命名5个物品中的2个）
  → 代表最近发展区
  → 干预从当前水平继续推进，不从零开始
  → 在 IEP 中优先于 0 分的项目
```

---

## 13. IEP 周期

```
IEP 周期：3 个月
短期目标预期：每个目标 2~4 周达成
评估频率：每月评估一次短期目标达成情况
```

---

## 14. 输出格式

```
| 阶梯及领域 | 任务目标 | 短期目标 |
|-----------|---------|---------|
| {年龄段}-{territory}-{subdivide} | {project.name} | {stage.name} |
```

导出文件：`{姓名}个别化教育计划.docx`

---

## 15. 确定性要求

- 同一份评估数据 + 同一个课包 → 同一份 IEP
- 所有项目名称和阶段名称必须与课包完全一致
- 不能编造、改写或合并课包中的项目
