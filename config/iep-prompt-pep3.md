# PEP-3 评估结果 → IEP 生成规则

> 本文档定义 PEP-3 评估结果如何转化为个别化教育计划（IEP）。
> 核心原则：IEP 中的所有干预项目必须来自指定课包的干预项目配置表，不能自行编造。

---

## 1. 评估数据结构

PEP-3 评估数据包含三部分：

| 板块 | CSV一级分类 | 评分体系 | IEP 用途 |
|------|-----------|---------|---------|
| 发展副测验 | 发展副测验 | 0=不通过, 1=萌芽, 2=通过 | 萌芽项 = 最近发展区，优先干预 |
| 行为副测验 | 行为副测验 | 0=严重, 1=轻微, 2=恰当 | 严重/轻微项 = 行为干预目标 |
| 照顾者报告 | 照顾者报告 | 多种评分 | 补充家庭信息，辅助目标制定 |

## 3. IEP 编写流程

```
Step 1：汇总评估数据
  输入：PEP-3 答题数据 + 儿童基本信息
  输出：各领域得分率、萌芽项清单、行为问题清单

Step 2：确定发展年龄
  规则：发展年龄（月）= 得分率 × 生理年龄（月）

Step 3：筛选薄弱领域（见第5节）
Step 4：确定干预年龄段（见第9节）
Step 5：匹配干预项目（见第10节）
Step 6：控制数量（见第8节）
Step 7：生成 IEP 表格
```

---

## 4. IEP 表格结构

| 列 | 数据来源 | 示例 |
|----|---------|------|
| 阶梯及领域 | `{年龄段}-{territory}-{subdivide}` | `2-3岁-沟通-语言理解` |
| 任务目标 | 干预项目 `name` | `简单句理解` |
| 短期目标 | 干预项目阶段 `stage.name` | `人物+动作/地点` |

---

## 5. 薄弱领域筛选

### 5.1 发展副测验

```
得分率 = P题数(得2分) / 总题数 × 100%（只计完全通过的题目占比）
薄弱领域 = 得分率 ≤ 65%
```

### 5.2 行为副测验

```
行为领域评级 = S(严重) / M(轻微) / A(恰当)
纳入 IEP = 评级为 S 或 M
```

### 5.3 照顾者报告

不直接生成 IEP 条目，但用于：
- 验证评估结果（不一致时优先采信测试员评估）
- 个人自理(PSC)薄弱项可补充到生活自理 IEP 目标
- 适应行为(AB)帮助判断哪些目标可在家庭泛化

---

## 6. 优先级排序

```
P0：行为副测验中评级为 S 的领域（严重行为问题优先处理）
P1：发展副测验得分率 ≤ 35% 且萌芽项 ≥ 2 个的领域
P2：发展副测验得分率 ≤ 35% 的领域
P3：发展副测验得分率 36%~50% 且萌芽项 ≥ 2 个的领域
P4：发展副测验得分率 36%~50% 的领域
P5：发展副测验得分率 51%~65% 的领域
P6：行为副测验中评级为 M 的领域
P7：照顾者报告中个人自理得分率 ≤ 50% 时，补充生活自理目标
```

## 8. IEP 项目数量控制

### 8.1 领域和目标数量

| 项目 | 推荐 | 上限 |
|------|------|------|
| 干预领域数 | 3~5 个 | 6 个 |
| 每领域任务目标 | 2~4 个 | 5 个 |
| 每目标短期阶段 | 1~3 个 | 全部阶段 |
| 整份 IEP 总目标 | 8~15 个 | 20 个 |

原则：宁可少而精，不可多而散。

### 8.2 按发展年龄调整

| 发展年龄 | 领域数 | 总目标数 | 说明 |
|---------|--------|---------|------|
| < 2岁 | 2~3 | 6~8 | 低龄注意力短，目标少而集中 |
| 2~3岁 | 3~4 | 8~12 | 逐步增加覆盖 |
| 3~5岁 | 4~5 | 10~15 | 学龄前关键期 |
| > 5岁 | 3~5 | 8~12 | 聚焦社交、学业准备、生活自理 |

### 8.3 特殊情况

**全面落后（所有领域 ≤ 35%）**：优先选沟通 + 认知(模仿) + 社交情绪 3个基础领域。

**能力不均衡（强弱差距 > 30%）**：弱势领域正常纳入，强势领域标注"可作为教学媒介"。

---

## 9. 年龄段匹配

**按发展年龄选课包年龄段，不按生理年龄。**

| 领域发展年龄（月） | 课包年龄段 |
|-----------------|---------|
| 0~23 | 1-2岁 |
| 24~35 | 2-3岁 |
| 36~47 | 3-4岁 |
| 48~59 | 4-5岁 |
| ≥ 60 | 5岁以上 |

边界处理：发展年龄在边界附近时，同时查看相邻两个年龄段，优先选较低年龄段中未掌握的项目。

---

## 10. 干预项目匹配规则

### 10.1 匹配流程

```
对每个薄弱领域：
  1. 通过领域映射表（第7节）→ 得到课包 territory + subdivide
  2. 筛选：该 territory + subdivide + 干预年龄段 的所有干预项目
  3. 排序匹配：
     a. 与萌芽技能直接对应的干预项目（最优先）
     b. 与不通过技能对应的干预项目中，阶段最低的
     c. 该领域该年龄段的其他干预项目（补充）
  4. 取前 2~4 个作为该领域的任务目标
```

### 10.2 阶段选择

```
每个干预项目的阶段按编号顺序排列：
  → 短期目标 = 从阶段 1 开始列出
  → 如果评估显示儿童已具备某阶段能力，可跳过
  → 每个任务目标列 1~3 个阶段
```

### 10.3 匹配失败

```
课包中找不到对应干预项目时：
  → 跳过，不生成 IEP 条目
  → 在备注中标注"XX领域建议补充专项干预方案"
  → 绝对不能编造不存在的项目名称
```

---

## 11. 萌芽技能处理

萌芽(1分)项是 IEP 的核心切入点——代表儿童"快要会但还不稳定"的能力（最近发展区），干预从这里入手成功率最高、见效最快。

```
对每个薄弱领域中评为"萌芽"的项目：
  1. 在课包中查找与该技能最接近的干预项目
  2. 干预起始阶段 = 该技能当前水平的下一阶段（不从零开始）
  3. 优先于"不通过"项目列入 IEP
```

---

## 12. IEP 周期

```
IEP 周期：3 个月
短期目标预期：每个目标 2~4 周达成
评估频率：每月评估一次短期目标达成情况
```

---

## 13. 输出格式

```
| 阶梯及领域 | 任务目标 | 短期目标 |
|-----------|---------|---------|
| {年龄段}-{territory}-{subdivide} | {project.name} | {stage.name} |
```

导出文件：`{姓名}个别化教育计划.docx`

---

## 14. 确定性要求

- 同一份评估数据 + 同一个课包 → 同一份 IEP
- 所有项目名称和阶段名称必须与课包完全一致
- 不能编造、改写或合并课包中的项目
